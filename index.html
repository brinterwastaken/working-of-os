  • Operating Systems make our tasks simpler by abstracting the hardware from the user.
  • They also perform tasks like process management and memory management to efficiently use the computer hardware.
  • Many Operating Systems have APIs to allow developers to easily make applications, and to also directly access the hardware if required.
  • The OS Subsystem communicates with the Kernel of the OS using system calls.

Step 1: CPU Initialization

When the CPU receives power, it begins executing the firmware (BIOS/UEFI). The BIOS/UEFI runs Power-On Self Test (POST) to check hardware integrity.

Step 2: Loading and Executing Boot Loader

The firmware locates the bootloader on the storage drive, loads it into memory and executes it.

The bootloader then loads the operating system kernel into memory.

Step 3: Kernel Initialization and Startup Services

The kernel initializes system components and hardware drivers, then starts the initial process.

The init process runs startup scripts and services, allowing the computer to access network, GPU, and other devices/functions.

Step 4: Login Manager and Startup Applications

The login manager displays the login screen to the user.

When the user logs in, the OS starts the desktop environment and startup applications.

Functions of an OS

Functions of an OS

Some general functions of an OS are listed below.

  • Memory Management: The OS performs memory management functions to make sure memory is used effectively and the os runs smoothly.
  • Process Management: The OS manages resorces given to each process, to allow all processes to run efficiently.
  • File Management: The OS manages the filesystem and files stored on various devices, allowing easy read and write of files.
  • Device Management: The OS manages devices connected to the computer, and loads the appropriate drivers for each device.
  • Security and Access Control: The OS uses user authentication, and user-based permissions to prevent unauthorized access, and also has anti-virus software to protect against malware.

User Interface

User Interface

  • The user interface is one of the most important parts of a consumer OS.
  • Graphical User Interface (GUI) allows the user to easily interact with the computer with a keyboard and a mouse, and perform various tasks, with the output displayed on a screen.
  • To have windows displayed on a screen, the OS requires a system software called the window manager.
Window Manager

Window Manager

  • The window manager receives information of what has to be rendered from the application, and renders the window in the appropriate location on the screen.
  • It also manages moving of windows, minimizing, fullscreen, automatic tiling, and virtual desktops.
  • The window manager also has a compositor to add effects like blur and animations when opening and closing windows.
  • The window manager, taskbar, and other UI components (notifications, etc.) make up the Desktop Environment of the OS.

Major Operating Systems

and their use cases

Most operating systems running on Non-Embedded Systems can be classified into three types - Windows-(based?), Unix-based, Linux-based.

  • Unix and Linux share similar file structures, shells, and architectures, whereas Windows differs slightly in these aspects.
  • In fact, the MacOS operating system is based on the Unix kernel (darwin). The similarities between Linux and MacOS are evident based on the directory structure, commands, etc.
  • MacOS and Windows are widely used in consumer computers, as they are easy to use, reliable, and have great software support.
  • Many programmers choose Linux based Operating Systems for speed and efficiency, and to customize as per their requirements.
  • Linux can be customized exactly according to the use case, and hence it is widely used in servers, super computers, and IOT devices.
  • Users can also use a virtual machine to test software on another operating system without installing it to their own computer.
